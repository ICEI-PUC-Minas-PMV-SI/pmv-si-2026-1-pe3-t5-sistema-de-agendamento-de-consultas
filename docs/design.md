## 4.3 Protótipos das Interfaces

### Protótipo login

### 1. Objetivo da tela
A tela de login permite que o usuário acesse o sistema OdontoFlow por meio de autenticação com e-mail e senha previamente cadastrados. Ao clicar no botão “Avançar”, o sistema valida as credenciais e concede acesso caso estejam corretas. A tela também oferece as opções “Esqueci minha senha” e “Criar conta”, permitindo recuperação de acesso ou redirecionamento para cadastro de novos usuários.

<img width="1366" height="768" alt="Login" src="https://github.com/user-attachments/assets/acb3a334-206e-4d00-9ae7-cde7837c5f46" />

---

### 2. Aplicação dos Princípios Gestálticos

**Proximidade:** Os campos de e-mail e senha estão posicionados próximos, formando um grupo lógico de entrada de dados. As opções “Esqueci minha senha” e “Criar conta” estão agrupadas abaixo, indicando ações secundárias relacionadas.

**Similaridade:** Os campos de entrada possuem o mesmo formato visual (bordas, tamanho e cor), facilitando a identificação como elementos do mesmo tipo. Os botões seguem um padrão visual consistente, com destaque para o botão principal.

**Figura-fundo:** O formulário está dentro de um container claro que se destaca do fundo azul escuro, direcionando a atenção do usuário para a área interativa.

**Continuidade:** Os elementos seguem uma organização vertical (título → campos → ações), guiando o usuário de forma natural no fluxo de interação.

**Foco:** O botão “Avançar” possui maior destaque visual (cor mais forte e posição estratégica), indicando claramente a ação principal da tela.


---

### 3. Regras de Ouro

**Consistência:** Elementos seguem padrão visual uniforme.

**Feedback imediato:** Interface responde ao foco e interação do usuário.

**Prevenção de erros:** Campos organizados reduzem preenchimento incorreto.

**Redução da carga de memória:** Informações visíveis e intuitivas.

**Controle do usuário:** Opções claras (entrar, recuperar senha, criar conta).

**Fechamento de ciclo:** Login leva a uma próxima etapa clara do sistema.


---

### 4. Recomendações Ergonômicas

**Layout simples e limpo:** Evita sobrecarga cognitiva.

**Botões destacados:** Facilita identificação da ação principal.

**Espaçamento adequado:** Melhora legibilidade e interação.

**Uso de cores contrastantes:** Auxilia na visualização dos elementos.

**Facilidade de navegação:** Usuário entende rapidamente o que fazer.


---

### Protótipo cadastro

### 1. Objetivo da tela
A tela de cadastro permite que novos usuários criem uma conta no sistema OdontoFlow, informando nome, e-mail e senha. Ao clicar no botão “Avançar”, os dados são registrados no sistema. A opção “Já possuo uma conta” permite retornar à tela de login.

<img width="1366" height="768" alt="Cadastro" src="https://github.com/user-attachments/assets/29213959-4b90-4fff-97d3-e5a45b82a6a5" />

---

### 2. Princípios Gestálticos

**Proximidade:** Campos agrupados formam um bloco de cadastro.

**Similaridade:** Campos seguem o mesmo padrão visual.

**Figura-fundo:** Container claro destaca o formulário do fundo escuro.

**Continuidade:** Organização vertical orienta o preenchimento sequencial.

**Foco:** Botão “Avançar” é o principal ponto de ação.  

---

### 3. Regras de Ouro

**Consistência:** Mantém o mesmo padrão da tela de login.

**Feedback imediato:** Campos indicam interação ao usuário.

**Prevenção de erros:** Estrutura simples reduz falhas.

**Redução da carga de memória:** Apenas campos essenciais são exibidos.

**Controle do usuário:** Permite retornar ao login facilmente.

**Fechamento de ciclo:** Cadastro leva ao próximo fluxo do sistema.

---

### 4. Recomendações Ergonômicas

**Interface objetiva:** Foco direto na tarefa de cadastro.

**Campos bem distribuídos:** Facilita preenchimento.

**Boa legibilidade:** Tipografia clara e adequada.

**Ações bem definidas:** Botão principal e link secundário diferenciados.

**Baixa carga cognitiva:** Usuário não precisa interpretar elementos complexos.

## 4.4 Testes com Protótipos
Nesta seção você deve apresentar os testes realizados com usuários utilizando os protótipos de alta fidelidade desenvolvidos na seção anterior. O objetivo é avaliar a usabilidade, a clareza das informações e a adequação do design às necessidades das personas definidas no projeto.

Cada integrante do grupo deverá aplicar o teste com um usuário distinto, preferencialmente alinhado ao perfil das personas criadas. Devem ser definidas previamente as tarefas que o usuário deverá executar no protótipo (por exemplo: realizar um cadastro, buscar um produto, concluir uma compra).

Durante a aplicação do teste, registre observações sobre comportamentos, dúvidas, erros e comentários feitos pelo usuário, bem como o tempo necessário para a execução de cada tarefa. Ao final, colete o feedback do participante, destacando pontos positivos e aspectos a serem melhorados.

Os resultados obtidos por todos os integrantes devem ser consolidados, apresentando uma análise geral com os principais problemas encontrados, oportunidades de melhoria e as ações previstas para o projeto final.
